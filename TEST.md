## 一、基础流程测试

### TC-01 正常双缓冲读取（18 步全流程）

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 配置：双缓冲=开，页缓存=关，故障=无，路径=`/home/user1/notes.txt`，用户=`user1`，字节=`16384`，地址=`0x10000000` | |
| 2 | 点击「连接并初始化」 | ① 顶部徽章变 `⬇ IRP ↓ 下行请求`（青蓝色）② 「↻ 重置 (保留缓存)」按钮出现 ③ 连接状态变 `● Connected` |
| 3 | 逐步执行 L0 用户层（4 步） | 日志 `[S1 (1/4)]` 库函数调用 → `[S2 (2/4)]` fd 表查找 → `[S3 (3/4)]` access_ok → `[S4 (4/4)]` 陷入内核。L0 卡片显示 `1/4 步` ~ `4/4 步` |
| 4 | L1 VFS 层（5 步） | 路径解析 → ACL 通过 → 页缓存禁用(直读) → 缓冲分配 → IRP 构造下发 |
| 5 | L2 驱动层（3 步） | IRP 解析 → 寄存器编程 → 进程 BLOCKED。进程状态面板变「BLOCKED」橙色 |
| 6 | L4 硬件层（3 步） | **DMA 面板更新**：源地址 `0x0400: Disk0 Sector`，目标地址 `0xBEEF1000: Kernel Buf1`，传输字节 `16384`，状态 `SETUP` → `TRANSFERRING` → `DONE`。CMD/STS 寄存器更新 |
| 7 | L3 中断层（3 步） | ① 阶段徽章变 `DATA ↑ 数据返回`（钴蓝色，脉冲动画）② IRQ↑、DATA↑ 连线高亮 ③ 进程状态变 RUNNING ④ I/O 完成，文件偏移量 `0 → 16384` |
| 8 | 完成后 | ① 徽章变 `✅ 完成`（绿色）② 用户缓冲区显示数据 ③ 错误控制台绿色 `SUCCESS` ④ 共 18 步日志 |
| 9 | 再点 3 次「下一步」 | **日志不增加**（已完成的模拟忽略多余请求） |
